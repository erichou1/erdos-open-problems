A transparent, automated log of large-language-model attempts at the open problems on erdosproblems.com. Each entry shows a completeness score — how much of the argument the model actually established, not how confident it sounds — and links to the full reasoning transcript. Problems not yet attempted are listed too, so you can see what's left.

How this works. Problems are investigated under an offline, statement-locked research workflow with independent exploration and layered adversarial review. The completeness score measures how much of the argument was rigorously established. These are machine attempts, not peer-reviewed proofs — even candidate-proved is not a solved claim. Only the separate verification gate may promote a result. The Verdict dropdown is a human likelihood rating (High / Medium / Low); tick Checked once a person has read it, Fable once it's been formalised in Fable, and Works if it holds up — and leave Notes. These sync to the repo for everyone when you Connect GitHub (a fine-grained token with Contents: Read & write on this repo only; stored only in your browser, never committed). Without a token they persist locally.
