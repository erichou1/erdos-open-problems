What is the size of the largest $A\subseteq \{1,\ldots,N\}$ such that all sums $\sum_{n\in S}\frac{1}{n}$ are distinct for $S\subseteq A$?
#321: [ErGr80,p.43]
number theory | unit fractions
The open status of this problem reflects the current belief of the owner of this website. There may be literature on this problem that I am unaware of, which may partially or completely solve the stated problem. Please do your own literature search before expending significant effort on solving this problem. If you find any relevant literature not mentioned here, please add this in a comment.
Comment activity that has not yet been incorporated into the remarks
There are no solutions, partial or complete, claimed in the comments.
Let $R(N)$ be the maximal such size. Results of Bleicher and Erdős from [BlEr75] and [BlEr76b] imply that\[\frac{N}{\log N}\prod_{i=3}^k\log_iN\leq R(N)\leq \frac{1}{\log 2}\log_r N\left(\frac{N}{\log N} \prod_{i=3}^r \log_iN\right),\]valid for any $k\geq 4$ with $\log_kN\geq k$ and any $r\geq 1$ with $\log_{2r}N\geq 1$. (In these bounds $\log_in$ denotes the $i$-fold iterated logarithm.)

See also [320].

  • It is a coincidence by the fact that $20$ was exactly small enough that the sum of reciprocals being equal implied that a short (length 1) sum of reciprocal needed to be involved.

    If equality would have been true for $n=21$, then there should have been a subset of $[1..20]$ of size $15$ which could be extended by $21.$
    Since equal sums of reciprocals do not involve primes between $n/2$ and $n$, these can be neglected in the computation.
    Then a bruteforce verification (added on the github link) can do the job.

    For this, note that e.g. $1+\frac 1{21} = \frac1 2 + \frac 13+ \frac 17 + \frac 1{14}$
    and $\frac 16 + \frac 1{21} = \frac 17+ \frac{1}{14}$,
    so there were multiple crucial equalities of (longer) sums of reciprocals involving $21$.
    On the other hand, $\frac 1a=\frac 1b + \frac1{21}$ has no solution where $0<a<b<21$.

    The first "most wanted" is caught as such. Here "dead" (false) is less interesting than "alive".

    (with that idea, it only has been checked that $R(20+x)=14+x$ for $1 \le x \le 3$ without much effort)

  • Using linear programming, I also got the following values.

    $R(21+x)=15+x$ for $0 \le x \le 2$
    $R(24+x)=17+x$ for $0 \le x \le 3$
    $R(28+x)=20+x$ for $x \in \{0,1\}$,
    $R(30+x)=21+x$ for $0\le x \le 2$,
    $R(33+x)=23+x$ for $0\le x \le 1$ and
    $R(35+x)=24+x$ for $0 \le x \le 4.$

    So the sequence starts with
    $$1, 2, 3, 4, 5, 5, 6, 7, 8, 9, 10, 10, 11, 12, 12, 13, 14, 14, 15, 15,15,\\16,17,17,18,19,20,20,21,21,22,23,23,24,24,25,26,27,28.$$

    So for small values, $R(n+1)=R(n)$ if $n+1$ is quite smooth, with some exceptions such as $36.$
    (Also, one can have combinations with $p,2p, \ldots, kp$ that do not have $p$
    in the denominator for $k$ (polylog) quite small wrt to $p$, is another reason why drawing conclusions is hard)

    $\textbf{Idea efficient programming}$
    The idea was to first find all pairs $(A,B)$ of disjoint subsets of $[n]$ for which the sum of reciprocals is equal.
    Then $A \cup B$ cannot belong to a maximal set with the desired property.
    Since $\frac 1{p^k}$ cannot be equal to $\sum_B 1/b$ when $v_p(b)<k$ for every $b \in B$,
    and $\frac 1p \pm \frac 2p$ also has a factor $p$ in the denominator,
    we know that $p,2p$ for $p> \frac n3$ and $16,32,27,25$ never belong to such sets $A \cup B.$
    This made the computation of forbidden sets $A \cup B$ more efficient.
    (for $n=32,$ there were already $81949$ many of them, for $n=33$ there were $245435$, for $n=35$ around $700k$ and for $n=36$ there were $2302966$, indicating that it becomes hard to compute; 2 hours with a common pc for $n=36$)
    The program has been added for $n=36$ in https://github.com/teorth/erdosproblems/issues/161.
