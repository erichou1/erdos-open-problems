Is there an infinite set $A\subset \mathbb{N}$ such that for every $a\in A$ there is an integer $n$ such that $\phi(n)=a$, and yet if $n_a$ is the smallest such integer then $n_a/a\to \infty$ as $a\to\infty$?
#51: [Er95][Er98]
number theory

There are no solutions, partial or complete, claimed in the comments.
Carmichael has asked whether there is an integer $t$ for which $\phi(n)=t$ has exactly one solution. Erdős has proved that if such a $t$ exists then there must be infinitely many such $t$.

See also [694].

This is discussed in problems B36 and B39 of Guy's collection [Gu04].

    • This proof is wrong - it says that the smallest $n$ such that\[\phi(n) = \prod_{i=1}^k (p_i-1)\]is $n=\prod_{i=1}^kp_i$. This fails already for $k=2$ - for example $\phi(3)=\phi(2\cdot 3)$. (There is an obvious problem in that one can just divide by $2$ without changing the value of $\phi(n)$. But even excluding this there are problems, such as\[\phi(3\cdot 5\cdot 7) = \phi(5\cdot 13)\]and $13<3\cdot 7$.)

  • Most likely the answer is negative, i.e., if $n$ is minimal amongst all numbers with a given value of $\phi(n)$, then $n/\phi(n)$ must be bounded (or equivalently, $\sum_{p|n} 1/p = O(1)$). Intuitively, if $n$ has too many prime factors, it should be possible to swap out some prime factors for others to reduce $n$ while maintaining exactly the same value of $\phi(n)$. But it looks very difficult to prove this rigorously with our current analytic number theory technology!

    TerenceTao — 19:04 on 10 Aug 2025

    • This is a good heuristic. Just a minor refinement: such an argument should prevent n from having many *small* prime divisors. We cannot expect to prevent n from having many prime factors of any size. Namely, if there were infinitelly many Fermat primes, then their products (like $n=3*5*17*257$) would be good examples of such n. Luckily, such large primes do not contribute much to the sum of reciprocals, of course.
