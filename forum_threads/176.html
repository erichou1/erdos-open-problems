Let $N(k,\ell)$ be the minimal $N$ such that for any $f:\{1,\ldots,N\}\to\{-1,1\}$ there must exist a $k$-term arithmetic progression $P$ such that\[ \left\lvert \sum_{n\in P}f(n)\right\rvert\geq \ell.\]Find good upper bounds for $N(k,\ell)$. Is it true that for any $c>0$ there exists some $C>1$ such that\[N(k,ck)\leq C^k?\]What about\[N(k,2)\leq C^k\]or\[N(k,\sqrt{k})\leq C^k?\]
#176: [Er65b][Er73][Er74b][Er75b][ErGr79][Er80,p.91][ErGr80,p.15]
additive combinatorics | arithmetic progressions | discrepancy

There are no solutions, partial or complete, claimed in the comments.
When $\ell=k$ this is the van der Waerden number $W(k)$ (see [138]). Spencer [Sp73] has proved that if $k=2^tm$ with $m$ odd then\[N(k,1)=2^t(k-1)+1.\]Erdős and Graham write that 'no decent bound' is known even for $N(k,2)$.

Erdős [Er63d] proved that, for every $c>0$,\[N(k,ck)> (1+\alpha_c)^k\]where $\alpha_c\to 0$ as $c\to 0$ and $\alpha_c\to \sqrt{2}-1$ as $c\to 1$.

Hunter in the comment section observes that the local lemma implies an improved bound of\[N(k,ck) \gg \frac{2^k}{k^{O(1)}\sum_{i>\frac{1+c}{2}k}\binom{k}{i}},\]so that in particular as $c\to 1$ we have, for all large $k$, $N(k,ck)\geq (2-o(1))^k$ (where the $o(1)$ term $\to 0$ as $c\to 1$).

  • I made an attempt at a Lean formalization of a proposed polynomial upper bound for the Problem 176 quantity $N(k,\sqrt{k})$.
    The formalized bound is the following: For every $k \ge 2$, $N(k,\sqrt{k}) \le \left\lfloor {4 \over 3}(k^2-k+1)(k-1)(k(k-1)+1) \right\rfloor + 1$.
    In particular, $N(k,\sqrt{k}) = O(k^5)$, so this gives a direct exponential-type bound $N(k,\sqrt{k}) \le C^k$ for a suitable absolute constant $C$.

    Lean4web: https://live.lean-lang.org/#url=https%3A%2F%2Fraw.githubusercontent.com%2FKitaKen1%2Ferdos176-nksqrtk-polynomial-bound-lean%2Fmain%2Ferdos176_sqrt_lean4web.lean
    GitHub: https://github.com/KitaKen1/erdos176-nksqrtk-polynomial-bound-lean
    The local Lean checks currently pass with no sorry, admit, axiom, or unsafe in the project Lean files. The Lean file includes #check and #print axioms lines for the final theorem. The reported axioms are the standard Lean/Mathlib foundation set: [propext, Classical.choice, Quot.sound]

    The case $k=1$ is trivial: every one-term arithmetic progression has signed sum $\pm 1$, so $|S|=1=\sqrt{1}$ and $N(1,\sqrt{1})=1$.

  • I made an attempt at a Lean formalization of a proposed polynomial upper bound for the Problem 176 quantity $N(k,2)$.

    The Lean-checked final target is, for every $k \ge 5$, $N(k,2) \le \left\lfloor \dfrac{2(k^2-1)(k-1)(2k+1)}{k-3} \right\rfloor + 1$.
    GitHub: https://github.com/KitaKen1/erdos176-nk2-polynomial-bound-lean

    Lean4web: Lean4web live check
    The Lean file includes #check and #print axioms lines for the final theorem. The reported axioms are the standard Lean/Mathlib foundation set: [propext, Classical.choice, Quot.sound].

    The small cases are handled separately: for $k=1$ there is no finite $N(1,2)$ in this AP-sum formulation, and for $k=2,3,4$ the finite Boolean checks give $N(2,2)=3$, $N(3,2)=9$, and $N(4,2)=13$. The corresponding Lean files and Lean4web links are in the GitHub README appendix.
    In particular, this gives the original $C^k$-type upper-bound statement for the finite-valued range $k \ge 2$: the Lean theorem gives an $O(k^3)$ bound for $k \ge 5$, and any fixed polynomial is eventually bounded by $C^k$ for some constant $C > 1$; the remaining finite cases $k=2,3,4$ can be absorbed by increasing $C$ once.

  • Building on Adenwalla's parity observation above (that N(k,2) = N(k,1) for even k), we compute the first exact values of N(k,2) for odd k ≤ 11 by exhaustive enumeration (k ≤ 9) and SAT solver (k = 11):

    N(3,2) = 9, N(5,2) = 22, N(7,2) = 49, N(9,2) = 65, N(11,2) = 112.

    All satisfy N(k,2) ≤ k². We conjecture this holds for all k ≥ 2, which would answer Erdős's question N(k,2) ≤ C^k with a polynomial bound.

    Per-position entropy H_k(N) = log_2 c_k(N) / N of the feasible coloring set is monotonically non-increasing for every tested odd k from N = k+1 onward, suggesting an entropy-decrement route to N(k,2) = O(k²).

    Full paper with proofs and DIMACS files: https://doi.org/10.5281/zenodo.20763838

  • Another observation is that $N(k+1, l-1) < \frac{k}{k-1} N(k,l)$ as there is some $k$ term progression $P$ in $[1,N(k, l)]$ such that summing along $P$ has absolute value $\geq l$. Then the $k+1$-th term of $P$ is at most $\frac{k}{k-1}N(k, l)$ and reduces the absolute value by at most $1$.

    Based on my earlier comment, if $k\equiv l\mod 2$ then $N(k+1, l-1) < \frac{k}{k-1} N(k,l)=\frac{k}{k-1} N(k,l-1)$.

  • If $P$ is a $k$-term arithmetic progression, then $|\sum_{n\in P}f(n)|\equiv k\pmod 2$. Therefore, if $k\not\equiv l\pmod 2$ and $|\sum_{n\in P}f(n)|\geq l$, then $|\sum_{n\in P}f(n)|\geq l+1$. It follows that if $k\not\equiv l\pmod 2$ then $N(k,l+1)=N(k,l)$.

    EDIT: Therefore for even $k=2^tm$, we have $N(k,2)=N(k,1)= 2^t(k-1)+1$.

  • This bound from [Er63d] is out-dated. An application of the local lemma will show that $N(k,ck) \gg \frac{2^k}{k^{O(1)} \sum_{i\ge (1/2+c/2)k}\binom{k}{i}}$. Thus $\alpha_c\to 1$ as $c\to 1$ (the intuition is while there are $N^2$ progressions to worry about, there are only $k^{O(1)}N$ progressions containing any integer).

    Any improvement beyond these simple random bounds would be interesting (and have applications!). One thing I have begun to wonder about is if you take $N$ to be prime, $s:=k/100$ (say), and then have the two color classes be $\{x: x^s \in [N/2]+N\cdot \mathbb{Z}\}$ and its complement.
