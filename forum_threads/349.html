OPEN This is open, and cannot be resolved with a finite computation.
For what values of $t,\alpha \in (0,\infty)$ is the sequence $\lfloor t\alpha^n\rfloor$ complete (that is, all sufficiently large integers are the sum of distinct integers of the form $\lfloor t\alpha^n\rfloor$)?
#349: [ErGr80,p.57]
number theory | complete sequences
The open status of this problem reflects the current belief of the owner of this website. There may be literature on this problem that I am unaware of, which may partially or completely solve the stated problem. Please do your own literature search before expending significant effort on solving this problem. If you find any relevant literature not mentioned here, please add this in a comment.

There are claims of partial results and/or a path to a solution in the comments.
Even in the range $t\in (0,1)$ and $\alpha\in (1,2)$ the behaviour is surprisingly complex. For example, Graham [Gr64e] has shown that for any $k$ there exists some $t_k\in (0,1)$ such that the set of $\alpha$ such that the sequence is complete consists of at least $k$ disjoint line segments. It seems likely that the sequence is complete for all $t>0$ and all $1<\alpha < \frac{1+\sqrt{5}}{2}$. Proving this seems very difficult, since we do not even know whether $\lfloor (3/2)^n\rfloor$ is odd or even infinitely often.

  • I have just uploaded my paper on this problem to the arXiv.

    Combined with (and relying on) the results from Graham's 1964 paper, the case $\alpha \ge \frac{1 + \sqrt{5}}{2}$ is completely settled. Moreover, even though my paper does not fully resolve the problem for the remaining interval $1 < \alpha < \frac{1 + \sqrt{5}}{2}$, I can at least give the full range of values of $t$ for which the resulting sequence is entirely complete, i.e. covers all positive integers.

    In the spirit of Graham, here you can find a plot for part of the 2D plane which shows where the sequence is and isn't (entirely) complete. To describe it in words, this is the current state of affairs:

    If $\alpha > 2$, the sequence is not complete for any $t > 0$.
    If $\alpha = 2$, the sequence is complete if, and only if, $t = \frac{1}{2^k}$ for some $k \ge 1$.
    If $5^{1/3} < \alpha < 2$ and $t \ge 1$, the sequence is not complete.
    If $5^{1/3} < \alpha < 2$ and $t < 1$, it's complicated (but essentially known). See Graham's paper or the above image for more details.
    If $\frac{1 + \sqrt{5}}{2} \le \alpha \le 5^{1/3}$, the sequence is complete if, and only if, $t < \min\left(\frac{3}{\alpha^2}, \frac{5}{\alpha^3} \right)$.
    If $1 < \alpha < \frac{1 + \sqrt{5}}{2}$, the sequence is entirely complete if, and only if $t < \min\left(\frac{2}{\alpha}, \frac{3}{\alpha^2} \right)$.
    If $1 < \alpha < \frac{1 + \sqrt{5}}{2}$ and $t \ge \min\left(\frac{2}{\alpha}, \frac{3}{\alpha^2} \right)$, the sequence might be complete without being entirely complete, but only partial results are known (see below).
    If $\alpha = 1$, the sequence is complete if, and only if, $1 \le t < 2$.
    If $\alpha < 1$, the sequence is not complete for any $t > 0$.

    As for those partial results for the region where it's not fully known yet, one can observe that for any compact set contained in that region, determining whether the sequence is complete for all pairs $(t, \alpha)$ in that set, in principle reduces to a finite search. I have therefore done some coding and can show the following computer-assisted result:

    If $\alpha \le 1.4$, the sequence is complete for all $t \le 3$.
    If $\alpha \le 1.3$, the sequence is complete for all $t \le 5$.
    If $\alpha \le 1.2$, the sequence is complete for all $t \le 10$.
    If $\alpha \le 1.1$, the sequence is complete for all $t \le 50$.

    These ranges are mostly a proof of concept, and for anyone interested it shouldn't be too hard to extend them further.

    The final result I prove in my paper is that the sequence is complete whenever $1 < \alpha \le 1 + \frac{1}{\lceil t \rceil + 2 \lceil \sqrt{t} \rceil}$, which (when we plot the values of $t$ and $\alpha$) is the first result that implies the completeness of the sequence for a region of infinite area!

  • Suppose we take $b_k$ to be the deduplicated increasing tail of $\lfloor t\alpha^n\rfloor$, and look at its subset sums. If for some $K_0$ there is an interval $[A,A+L]\subseteq\Sigma_{K_0}$ with cardinality $L+1\ge b_{K_0+1}$, then once the eventual overlap $b_{m+1}\le 2b_m-1$ (which eventually holds for large $m$ when $1<\alpha<2$, up to sporadic floor effects, and even $b_{m+2}\le b_{m+1}+b_m-1$ for $1<\alpha<\varphi$) has started to hold, the translate–overlap mechanism forces that from some point onward all sufficiently large integers are covered. So for $\alpha<\varphi$ it seems the whole problem reduces to finding one finite seed interval beyond the eventual regime.

    What I am not sure about is whether this reduction is completely sound, or whether there could be a gap before the overlap inequality takes effect. Do we know that some seed must appear for every $t>0$ and $1<\alpha<\varphi$, or could it in principle fail entirely in the finite pre-asymptotic window? The seed condition I think has to be expressed in cardinalities ($L+1\ge b_{K_0+1}$), otherwise there is an off-by-one issue. Deduplication is required too, since the problem is about distinct integers. Is there an argument that guarantees a seed in all cases, or is this the open difficulty? If the latter, what kinds of computational (python) directions or heuristics might probe at guaranteeing the seed interval (e.g. generate sequence -> compute sums -> detect first covering interval)? Thank you.

    • Proving that such a seed always exists is essentially the difficulty, yes. Furthermore, you are correct that -as stated- the problem requires distinct integers of the form $\left \lfloor t \alpha^n \right \rfloor$, but that is not what Erdős and Graham wrote (or what I believe they intended), and also not what is assumed in Graham's 1964 paper. Every element $s_n = \left \lfloor t \alpha^n \right \rfloor$ is allowed to be used once, even if $s_n = s_m$ for some $m \neq n$. For example, Theorem 2 in Graham's paper implies in particular that the sequence with $t = \frac{2}{3}$ and $\alpha = 1.7$ (which has $s_1 = s_2 = 1$ and $s_3 = 3$) is entirely complete, which would not be true if we are not allowed to use both $s_1$ and $s_2$. For what it's worth, I am currently writing up some slight extensions of Graham's results that I will hopefully link to in the coming week.

  • The script below shows that for $t=1,\alpha=1.5$ and $\alpha=\varphi$ all integers up to $200{,}000$ are representable, while for $t=2,\alpha=1.4$ a few small numbers ($1,4,6,\dots$) are missing even though the density is very high. It is only a computation, not a proof, but it hopefully illustrates the behaviour discussed in Problem 349.

    import math

    def seq(t, a, n=40):
    return [int(t * a**k) for k in range(1, n+1)]

    def cover(s, L=200000):
    r = [0] * (L+1)
    r[0] = 1
    for a in s:
    for x in range(L, a-1, -1):
    r[x] |= r[x - a]
    return sum(r[1:]) / L, [i for i in range(1, L+1) if not r[i]]

    for t, a in [(1, 1.5), (1, (1 + 5**0.5) / 2), (2, 1.4)]:
    s = seq(t, a)
    d, m = cover(s)
    print(f"\nt={t}, α≈{round(a, 3)}")
    print("First terms:", s[:12])
    print(f"Density up to 200k: {d:.6f}")
    print("First missing:", m[:10] if m else "None")


    We get:
