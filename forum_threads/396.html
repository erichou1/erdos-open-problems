Is it true that for every $k$ there exists $n$ such that\[\prod_{0\leq i\leq k}(n-i) \mid \binom{2n}{n}?\]
#396: [ErGr80]
number theory | binomial coefficients
The open status of this problem reflects the current belief of the owner of this website. There may be literature on this problem that I am unaware of, which may partially or completely solve the stated problem. Please do your own literature search before expending significant effort on solving this problem. If you find any relevant literature not mentioned here, please add this in a comment.
Comment activity that has not yet been incorporated into the remarks
There are no solutions, partial or complete, claimed in the comments.
Erdős and Graham write that $n+1$ always divides $\binom{2n}{n}$ (indeed $\frac{1}{n+1}\binom{2n}{n}$ is the $n$th Catalan number), but it is quite rare that $n$ divides $\binom{2n}{n}$.

Pomerance [Po14] has shown that for any $k\geq 0$ there are infinitely many $n$ such that $n-k\mid\binom{2n}{n}$, although the set of such $n$ has upper density $<1/3$. Pomerance also shows that the set of $n$ such that\[\prod_{1\leq i\leq k}(n+i)\mid \binom{2n}{n}\]has density $1$.

The smallest $n$ for each $k$ are listed as A375077 on the OEIS.

  • I looked at the theoretical side a little bit with GPT-5.4 Thinking.

    Let $E_k := \{ n : n(n-1)\dots(n-k) \mid \binom{2n}{n} \}$. Let $d$ denote density and $\overline{d}$ denote upper density.

    Pomerance proved $\overline{d}(E_0) < 1/3$. In fact his argument gave $\overline{d}(E_0) \le 1-\log 2 = 0.306\dots$. Nevertheless, Ford and Konyagin (2021) has completely settled this as $d(E_0) = c_1 := 0.114\dots$; in particular the density of the set exists.

    It follows trivially that $\overline{d}(E_k) \le c_1$. But we were not able to even prove $\overline{d}(E_k) < c_1$ for $k > 0$. However, heuristically we do expect $\overline{d}(E_k) \approx c_1^{k+1}$. This is confirmed to line up well with the available computational evidence. In particular, the smallest $n$ for each $k$ should increase by about $1/c_1 \approx 8.75$ each time.

      Let

      $$E_k := \left\{ n > k : \prod_{j=0}^k (n-j) \mid \binom{2n}{n} \right\},$$

      and let $\overline{\delta}$ denote upper logarithmic density.

      For any fixed $k \ge 1$, if $n \in E_k$, then in particular both $n \mid \binom{2n}{n}$ and $n-1 \mid \binom{2n}{n}$. Apply the same no-carry argument used by Pomerance: if $p \mid (n-j)$ with $p > \sqrt{2n}$, then $p \nmid \binom{2n}{n}$. Hence every $n \in E_k$ must satisfy

      $$P^+(n) \le \sqrt{2n} \qquad \text{and} \qquad P^+(n-1) \le \sqrt{2n},$$

      where $P^+$ is the largest prime factor.

      So $E_k$ is contained in the set of consecutive $\sqrt{2x}$-smooth pairs up to $x$. Recent work of Tao-Teräväinen on consecutive smooth numbers then gives, on a set of $x$'s of logarithmic density $1$,

      $$\#\left\{ m \le x : P^+(m) \le x^{1/u},\; P^+(m-1) \le x^{1/u} \right\} = (\rho(u)^2 + o(1))\,x.$$

      Taking $u \to 2$ gives

      $$\overline{\delta}(E_k) \le \rho(2)^2 = (1-\log 2)^2 \approx 0.09416.$$

      Since Ford–Konyagin proved

      $$d(E_0) = c_1 \approx 0.11424,$$

      this yields the strict inequality

      $$\overline{\delta}(E_k) \le (1-\log 2)^2 < c_1 \qquad (k \ge 1).$$

      So while this may not prove $\overline{d}(E_k) < c_1$ in the ordinary sense, it does show that once you ask for even one backward step, the set is already strictly sparser than the governor set at the logarithmic-density level. In particular, if the ordinary density $d(E_k)$ happens to exist, then automatically

      $$d(E_k) < c_1.$$

      At least to me, that seems significant: it says the $k > 0$ problem is not just "the $k = 0$ problem with a bounded shift" but a genuinely more correlated/rarer pattern problem.

    $\textbf{Exhaustive search, validation, and a formally verified barrier theorem}$

    Over the past several months I have been running an exhaustive computational search for [396] witnesses, extending OEIS A375077 with 6 new terms at the initial time of discovery (2 new terms at the time of this post). The full codebase, validation certificates, and a machine-checked proof are publicly available at github.com/jdehorty/erdos396.

    *Note: Dates reflect my own personal records of when each value was first found, not when results were submitted or published. The jumps in pace (particularly from k=11 onward) correspond to successive sieve improvements: the v1 prefilter (7x), the fused sieve pass (15x), and finally Kummer-based valuation for all primes (31x single-threaded throughput).

    Each value has a computational certificate of minimality: every integer below the claimed witness has been checked and ruled out.

    Note on $k=9$: the original governor-run search found a witness at $17{,}609{,}764{,}993$. A subsequent full validation pass (checking every integer, not just governor-run candidates) found the true minimum at $1{,}019{,}547{,}844$.

    $\textbf{Method.}$ The search is built around the Governor Set $G = \{n : n \mid \binom{2n}{n}\}$ (OEIS A014847, density $\approx 12.3\%$ by Ford-Konyagin). All known witnesses have every block term $n, n-1, \ldots, n-k$ in $G$. A fused segmented sieve strips small prime factors and checks the Kummer carry condition $\nu_p(n) \leq \nu_p\!\binom{2n}{n}$ inline, rejecting ${\sim}87\%$ of integers in a single pass. The $\sqrt{2n}$ large-prime-factor barrier provides additional fast rejection. Runs of $k+1$ consecutive governors are candidate witnesses, each verified by the full inequality $\sum_{i=0}^{k} \nu_p(n-i) \leq \nu_p\!\binom{2n}{n}$ at every prime $p$.

    $\textbf{The Small Prime Barrier Theorem.}$ The governor-run search cannot detect witnesses where some block term fails to be a governor. I proved that any such non-governor failure must occur at a prime $p < 2k+1$. For $k=13$ this is just 9 primes: $\{2,3,5,7,11,13,17,19,23\}$. The proof uses Kummer's theorem and a carry invariance lemma for base-$p$ self-addition; the bound $2k+1$ is tight (via Bertrand's postulate). The Small Prime Barrier Theorem and its tightness are formally verified in Lean 4 + Mathlib (no sorry/admit); the formal proof covers the carry invariance lemma, the barrier theorem itself, and the tightness result via Bertrand's postulate - not the computational search.

    This yields a provably complete validation method: screen every integer at the barrier primes, then fully verify survivors. The validate binary implements this for $k=8$ through $k=13$, producing minimality certificates independent of the Governor Set Completeness Conjecture.

    $\textbf{Verification.}$ The repository includes: (1) a Rust verifier and a stdlib-only Python verifier for witness validity; (2) the Lean 4 formal proof of the Small Prime Barrier Theorem (justifying the completeness of the validation algorithm); (3) published validation certificates with coverage invariants and build metadata tied to specific source revisions; (4) an independent report checker that recomputes invariants from the JSON reports. Full details at github.com/jdehorty/erdos396.

    Note: AI usage comprised GPT-5.4 Pro (extended thinking) and Google Gemini 3.1 DeepThink for proof exploration and adversarial review, and Claude Opus 4.6 for coding the Rust-based search pipeline and Lean 4 formalization. The Lean 4 formal proof is machine-checked (no sorry/admit), and all computational claims are backed by deterministic validation certificates; see docs/trust.md in the repository.

        • Some surprising updates - turns out sometimes there's some simple things that can achieve radical speedups. I've crossed the $2$ billion candidates/s barrier (at $k\in\{7,8,9\}$), with a gradual slowdown for $k \geq 11$.

          Some major speedups you might be interested in:
          1. Like you said, you can compute the multiplicative inverse $\pmod{2^{64}}$, which you can use Newton-Raphson iteration to do very quickly.
          2. In the final array you check for smoothness, you can use Boyer-Moore string-search algorithm to reduce the search *significantly*.
          3. You don't need to check every prime. Small primes you can let the compiler optimise for you, medium size primes you can do the same thing we always do, but for large primes, often the interval you're checking on doesn't have any such elements so you can skip them.
          4. If you're using multi-threading, do some profiling to see if your threads are being starved etc, there's some ways to shuffle around how they compute to minimise. Also be careful of threads stalling cuz they're trying to write out to a file. I found that I had a bunch of data getting streamed from L3 to L2, and I realised that my chunks I processed were just too big to fit in the L2 cache.
          5. Make sure you're not running other programs in parallel (my previous numbers were with me running VSCode and FireFox at the same time)
          In general, get a proper profiler to look at the code - I have been using Intel VTune Profiler, which is free but can't give me everything since I have an AMD Ryzen 7 processor - I tried using AMD uProf, but it would just crash whenever I ran a program.

    Extending A375077



    Using a prime-by-prime Kummer carry check (no direct computation of $\binom{2k}{k}$), I verified $a(1)$ through $a(7)$ and found:

    A necessary large-prime-factor bound


    Claim. If $\prod_{i=0}^{n}(K - i)$ divides $\binom{2K}{K}$ and $K > n$, then every prime $p > 2n$ dividing some $K - j$ satisfies $p \leq \sqrt{2K}$.

    Proof. Suppose $p > 2n$, $p > \sqrt{2K}$, and $p \mid (K - j)$ for some $0 \leq j \leq n$. Write $K = ap + j$. Since $p^2 > 2K$, we have $a < K/p < p/2$, so $K$ has at most two base-$p$ digits with leading digit $a < p/2$ and units digit $j \leq n < p/2$. Adding $K + K$ in base $p$: the units digit gives $2j < p$ (no carry), the leading digit gives $2a < p$ (no carry). By Kummer's theorem, $\nu_p(\binom{2K}{K}) = 0$. But $\nu_p(\prod(K - i)) \geq \nu_p(K - j) \geq 1$, so the divisibility fails. $\square$

    In other words:

    $$P^{+}\!\left(\prod_{i=0}^{n}(K - i)\right) \leq \max(2n, \lfloor\sqrt{2K}\rfloor).$$

    So any solution must have all prime factors of the descending block below the natural threshold $\sqrt{2K}$ (aside from the trivial small-prime range $p \leq 2n$).

    Empirical confirmation

    Connection to #728



    Problem #728 (arXiv:2601.07421) proved the existence of "carry-rich but spike-free" integers for ascending windows $m+1, \ldots, m+k$ using the fact that when $p > 2k$ and $p^r \mid (m+i)$, the low base-$p$ digits of $m$ are forced to be large, automatically producing carries (Lemma 5). For descending blocks $K, K-1, \ldots, K-n$, this mechanism reverses: $p \mid (K - j)$ forces low digits of $K$ to be small, so no carries are automatic. This is why the #728 argument does not directly prove $a(n) < \infty$.

    The large-prime-factor bound above makes this precise. The new obstruction appears to lie not in the small-prime Kummer analysis addressed in #728, but in controlling the large prime factors of the descending block. This aligns with Tao's observation (Sep 2025) that the initial difficulty is "securing a string of consecutive smooth numbers."

    This suggests a two-part structure:
    1. A small/medium-prime digit-pattern problem: this may be approachable by adapting the carry-rich ideas of #728.
    2. A large-prime-factor problem: any solution must satisfy $P^{+}(\prod_{i=0}^n (K-i)) \leq \max(2n, \lfloor\sqrt{2K}\rfloor)$, and #728 provides no mechanism for forcing this in descending blocks.
    For primes $p > 2n$ in the intermediate range $\sqrt{K} < p \leq \sqrt{2K}$, any divisor $p \mid (K - j)$ can contribute at most one carry on the Kummer side. For all known values checked so far, these primes are satisfied with exactly one carry and zero slack. So the "one-carry regime" appears to be the binding large-prime constraint in practice.

      ## A specific open subproblem

      After more work on the structure of the divisibility condition, one route I’ve been exploring leads to a medium-prime subproblem that now seems to be a main obstacle. I’m not claiming a full reduction here; I just want to state the subproblem in case it connects to something known.

      **Setup.** By Kummer,\[
      \prod_{j=0}^n (K-j)\mid \binom{2K}{K}
      \]iff at every prime \(p\), the carry count \(\kappa_p(K)\) satisfies\[
      \kappa_p(K)\ge \sum_{j=0}^n \nu_p(K-j).
      \]For large primes \((p > \sqrt{2K})\), the smoothness bound from my earlier post handles these.

      For small primes \((p \le Y\), with \(Y\) a fixed parameter depending on \(n)\), I have a plausible truncation/low-digit approach, but I do not yet regard that part as complete.

      The range where I am currently stuck is the **medium-prime range \(Y < p \le \sqrt K\)**.

      ## The medium-prime question

      After restricting to \(K\) with \(p^2 \nmid (K-j)\) for all \(p > Y\) and all \(j \le n\), one loses at most\[
      O_n\!\left(\sum_{p>Y}\frac{1}{p^2}\right)
      \]in density, so this loss can be made arbitrarily small by taking \(Y\) large. On this restricted set, any medium prime \(p > Y\) dividing some \(K-j\) has \(\nu_p(K-j)=1\). Since \(p > Y \ge n\), at most one \(j\) has \(p \mid (K-j)\). So the condition at such a prime is simply \(\kappa_p(K)\ge 1\): at least one carry when doubling \(K\) in base \(p\).

      Here is the key structural observation: if \(p > 2n\) and \(p \mid (K-j)\) for some \(0 \le j \le n\), then \(K \equiv j \pmod p\), so the base-\(p\) units digit of \(K\) is \(j \le n < p/2\). Doubling gives \(2j < p\), so there is no carry at the units position. Any carry must come from a higher-order digit.

      For primes near \(\sqrt K\) with only \(2\)–\(3\) base-\(p\) digits total, this is genuinely restrictive — the carry condition fails for a positive fraction of \(K\) in each residue class.

      Define\[
      f(K)=\#\{\,\text{primes } Y < p \le \sqrt K : \exists j \le n \text{ with } p \mid (K-j)\text{ and }\kappa_p(K)=0\,\}.
      \]A first-moment calculation suggests \(E[f(K)] = O_n(1)\); heuristically the dominant contribution comes from primes with \(3\) base-\(p\) digits.

      If the small-prime piece can be completed, then along this route it would remain to show \(P(f=0)>0\). A natural way to try to do this is via Poisson approximation methods (e.g. Stein–Chen), which would require adequate control on the pairwise covariances \(\operatorname{Cov}(1_{B_p},1_{B_q})\).

      For “easy” prime pairs with \(p^{L_p} q^{L_q} \ll X\), approximate independence should follow from CRT (the base-\(p\) and base-\(q\) digit constraints live on coprime moduli that fit inside the interval). But for primes \(p, q\) both in \((X^{1/3}, \sqrt X]\) — each with only \(3\) base-digits — the product \(p^3 q^3\) exceeds \(X\), so the interval contains fewer than one full CRT period, and I don’t see how to decouple the events.

      This hard-pair regime contributes a bounded but nonzero amount to the mean, so it cannot simply be discarded.

      **The concrete question.** For primes \(p, q \asymp X^{1/3}\) and a fixed integer \(\Delta\), let\[
      T_p = \{\,m \le X/p : \text{every base-}p\text{ digit of }m\text{ is }< \lceil p/2\rceil \,\}.
      \]What can be said about\[
      \#\{(a,b)\in T_p\times T_q : pa-qb=\Delta\}\,?
      \]This is a lattice point count on a product of digit-restricted sets under a linear constraint. Has this type of intersection problem appeared in the digital restrictions or additive combinatorics literature? Any pointers would be appreciated.

    • Partial progress on Problem 396: small n and a reduction for general n

      I've been working on this problem and have some partial results plus a precise formulation of what seems to be the remaining obstacle for general n. AI tools (Claude, GPT) were used throughout. The computations were checked directly, and the mathematical reductions below are the parts I currently trust most.

      Setup

      By Kummer's theorem, the divisibility condition reduces to: for every prime p, the sum of p-adic valuations of K, K−1, …, K−n must not exceed the number of carries when doubling K in base p.

      After a standard squarefree sieve (restricting to K with p² ∤ (K−j) for large primes), this simplifies for primes p > Y to: whenever p divides some K−j, we need at least one carry in the base-p addition K+K.

      Small primes

      For primes p ≤ Y, the carry count κ_p(K) grows with the number of base-p digits via a 2-state Markov chain with spectral gap (p−1)/p. The required valuation sum is bounded above by ν_p((n+1)!) (Legendre). I believe the right route is to combine this factorial bound with a carry-concentration argument in a fixed CRT progression, but I have not yet written this up as a formal standalone proof.

      Medium primes: the key structural observation

      For a medium prime p > Y dividing K−j, write K = j + pa and decompose the "bad" event B_p (all base-p digits of a are < ⌈p/2⌉, giving zero carries) as B_p = C_p ∩ M_p, where:

      - C_p restricts the higher digits of a (a "top-digit" condition)
      - M_p restricts the lowest digit: d₁ = a mod p < ⌈p/2⌉ (a "middle-digit" condition)

      Since B_p ⊆ M_p, avoiding all M_p events is sufficient for the medium-prime range.

      The event M_p is equivalent to K belonging to (n+1)⌈p/2⌉ forbidden residue classes modulo p² (these are distinct for p > n, by a short congruence argument). The local forbidden density is g(p) = (n+1)/(2p) + O(1/p²).

      CRT independence in the product space

      Since the moduli p² are pairwise coprime, the M_p events are exactly independent in the abstract CRT product space ∏ ℤ/p²ℤ. The density of the "good" set there is ∏(1 − g(p)), which by a Mertens-type evaluation is of order C_n/(log X)^{(n+1)/2} — positive but shrinking. The unresolved step is transferring that product-space density back to the interval [1, X].

      What works for small n

      For the medium-prime layer with L ≥ 3 base-p digits, the expected number of bad primes is E[g] = Σ_p P(B_p) ≈ 0.16(n+1). For n ≤ 5 this is less than 1, so Markov's inequality gives P(g = 0) > 0 directly — no sieve or independence argument needed for that layer. A complete proof integrating the small-prime and squarefree constraints with this would need a more careful writeup, but I believe no fundamental obstacle remains for small n.

      My more recent computations also show that bad-prime events in the L = 3 layer are positively correlated (pairwise ratio ≈ 2) rather than Poisson-independent, which is why I am not claiming a probabilistic closure for general n.

      What I'm stuck on for general n

      I need to show the "good" set in the CRT product space actually intersects [1, X]. The CRT product-space heuristic predicts a count of order X/(log X)^{(n+1)/2}, and direct computation confirms this for all tested n. But proving it requires one of:

      1. A sieve lower bound for {K ≤ X : K mod p² ∉ A_p for all medium p}. The Brun–Selberg sieve runs into trouble because ω(p) = (n+1)⌈p/2⌉ grows with p (remainder control fails for moduli p²), or because the sieve dimension κ = n+1 exceeds the sieving limit at s = 2 (for moduli p). I checked this against Friedlander–Iwaniec and the DHR higher-dimensional sieve; neither framework gives a positive lower bound in this parameter regime.

      2. A Fourier discrepancy estimate showing the CRT product forbidden set doesn't cluster in [1, X]. The per-prime Fourier transform of 1_{A_p} factors as S₁(k)·S₂(k) with explicit geometric-sum bounds, and CRT gives multiplicative factorization of hat(1_{A_d}). But controlling the low-frequency sum when d = ∏ p² ≫ X requires cancellation beyond what absolute-value bounds can provide; a CRT-profile analysis shows "enemy families" of low-frequency k where no per-prime Fourier saving is available.

      Computational evidence concerns only the L >= 3 medium-prime layer not the full sieve problem.

      P(g = 0) is stable and positive for all tested parameters:

      | n | Observed P(g=0), X = 100,000 | CRT product-space prediction (2/3)^{(n+1)/2} |

      | 1 | 0.668 | 0.667 |
      | 3 | 0.446 | 0.444 |
      | 5 | 0.298 | 0.296 |

      Has anyone encountered the problem of proving a lower bound for |{K ≤ X : K mod p² ∉ A_p for all primes Y < p ≤ √X}| where A_p has ~(n+1)p/2 forbidden classes mod p²? The CRT product space gives exact independence and a positive-density good set, but transferring this to [1,X] when ∏ p² ≫ X is the obstacle. Any pointers to relevant tools — perhaps from the sieve theory of "dense" forbidden sets, or from discrepancy theory for structured CRT products; would be very welcome.
